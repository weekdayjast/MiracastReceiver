## 📝 下一步操作

### 1. 配置环境变量（重要！）

运行提供的批处理脚本（以管理员身份）：

```bash
# 右键点击 -> "以管理员身份运行"
setup-android-env.bat
```

**或者手动配置**：
1. 打开"系统属性" → "高级系统设置" → "环境变量"
2. 新建系统变量 `ANDROID_HOME` = `D:\develop\android-sdk`
3. 编辑 `Path` 变量，添加：
   - `D:\develop\android-sdk\platform-tools`
   - `D:\develop\android-sdk\cmdline-tools\latest\bin`
   - `D:\develop\android-sdk\emulator`

### 2. 启动 Android Studio

```bash
# 打开 Android Studio
"C:\Program Files\Android\Android Studio\bin\studio64.exe"
```

**首次启动配置**：
- 选择 "Do not import settings"（如果是首次使用）
- SDK 路径设置为：`D:\develop\android-sdk`
- 完成安装向导

### 3. 创建投屏接收端项目

在 Android Studio 中：

1. **File → New → New Project**
2. 选择 **"Empty Activity"** 或 **"Android TV Activity"**
3. 项目配置：
   - Name: `MiracastReceiver`（或自定义名称）
   - Package: `com.yourname.miracast`
   - Language: **Kotlin**（推荐）或 Java
   - Minimum SDK: **API 21** (Android 5.0)
   - Target SDK: **API 36**

### 4. 验证开发环境

配置环境变量后，**重启终端**，运行以下命令：

```bash
# 验证 Java
java -version

# 验证 ADB
adb version

# 验证 SDK Manager
sdkmanager --version

# 列出已安装的 SDK 组件
sdkmanager --list | head -30
```

## 🔧 开发调试

### 使用真实 Android TV 设备

1. 在 TV 上启用开发者选项
2. 启用 USB 调试
3. 连接 USB 或使用 Wi-Fi ADB：

```bash
# 查看已连接设备
adb devices

# Wi-Fi 调试（TV 和电脑在同一网络）
adb connect <TV_IP_ADDRESS>:5555

# 安装 APK
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

### 使用 Android TV 模拟器

在 Android Studio 中：
1. Tools → Device Manager
2. Create Device
3. 选择 TV 类型（如 "Android TV 1080p"）
4. 选择系统镜像（API 36）
5. 启动模拟器

## 🐛 常见问题

### 问题 1: Gradle 同步失败
**解决方案**：配置国内镜像源

编辑 `settings.gradle.kts`：
```kotlin
pluginManagement {
    repositories {
        maven { url = uri("https://maven.aliyun.com/repository/gradle-plugin") }
        maven { url = uri("https://maven.aliyun.com/repository/google") }
        maven { url = uri("https://maven.aliyun.com/repository/public") }
        gradlePluginPortal()
        google()
        mavenCentral()
    }
}
```

### 问题 2: 模拟器无法启动
**解决方案**：
- 在 BIOS 中启用 VT-x/AMD-V
- 关闭 Hyper-V（与 HAXM 冲突）
- 安装 Intel HAXM

### 问题 3: ADB 无法识别设备
**解决方案**：
```bash
# 重启 ADB 服务
adb kill-server
adb start-server

# 检查设备
adb devices
```
