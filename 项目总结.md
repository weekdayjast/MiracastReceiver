## 技术亮点

### 1. 模块化设计
项目采用清晰的模块划分，各功能模块职责明确，便于维护和扩展。

### 2. 跨平台支持规划
通过 WebRTC 协议，可以轻松支持 Android、iOS、Windows、macOS 等多个平台的发送端。

### 3. 服务发现机制
使用 mDNS 实现局域网自动发现，无需手动输入 IP 地址，用户体验好。

### 4. 编解码能力自适应
自动检测设备支持的编解码器，选择最优配置，保证兼容性。

### 5. 前台服务设计
使用前台服务保证应用在后台持续监听投屏请求，不会被系统杀死。

## 构建说明

### 环境要求
- Android Studio 2026.1+
- Android SDK API 36
- JDK 17
- Gradle 8.11.1

### 构建步骤

```bash
# 1. 克隆或打开项目
cd D:\workspace\miracast\MiracastReceiver

# 2. 构建 Debug APK
./gradlew assembleDebug

# 3. 构建 Release APK
./gradlew assembleRelease

# 4. 安装到设备
./gradlew installDebug

# 5. 清理构建
./gradlew clean
```

### APK 输出位置
```
app/build/outputs/apk/debug/app-debug.apk
app/build/outputs/apk/release/app-release.apk
```

## 运行说明

### 在模拟器上运行
1. Android Studio → Tools → Device Manager
2. 创建 Android TV 设备
3. 选择系统镜像 API 36
4. 启动模拟器
5. Run → Run 'app'

### 在真机上运行
1. 电视上启用开发者选项
2. 启用 USB 调试或 Wi-Fi ADB
3. 连接设备：`adb connect <TV_IP>:5555`
4. 安装 APK：`adb install app-debug.apk`

## 已解决的问题

### 1. JDK 版本兼容性
**问题**: JDK 25 版本号格式导致 Kotlin 编译器报错
**解决**: 切换到 JDK 17

### 2. SDK 路径配置
**问题**: 构建时找不到 Android SDK
**解决**: 创建 `local.properties` 文件指定 SDK 路径

### 3. BuildConfig 未生成
**问题**: MiracastApp.kt 中 BuildConfig 类未找到
**解决**: 在 build.gradle.kts 中启用 `buildConfig = true`

### 4. compileSdk 版本警告
**问题**: AGP 8.7.3 官方支持到 compileSdk 35
**解决**: 添加 `android.suppressUnsupportedCompileSdk=36` 抑制警告
