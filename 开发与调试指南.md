# Miracast Receiver - 开发与调试指南

## 项目简介

这是一个运行在 Android TV 上的投屏接收端应用，支持 Android 手机、iPhone、Windows 和 Mac 电脑投屏到电视。

## 支持的投屏协议

- ✅ **AirPlay**（iPhone/iPad/Mac）- 完美支持
- ✅ **DLNA/UPnP**（各平台通用）- 完美支持
- ✅ **WebRTC 自定义协议**（自定义发送端）
- ⚠️ **Miracast/WFD**（Windows 无线显示器）- 受系统限制，需手动配对

## 构建与安装

### 环境要求

- Android Studio 2026.1+
- Android SDK API 36
- JDK 17+
- Gradle 8.11.1

### 打包 Debug APK

在项目 `MiracastReceiver` 目录下执行：

```bash
# Linux/Mac
./gradlew assembleDebug

# Windows
gradlew.bat assembleDebug
```

构建完成后，APK 位于：

```
app/build/outputs/apk/debug/app-debug.apk
```

### 打包 Release APK

```bash
./gradlew assembleRelease
```

**注意**：Release APK 需要签名才能安装。如果没有配置签名，会生成未签名的 APK：

```
app/build/outputs/apk/release/app-release-unsigned.apk
```

测试建议直接使用 Debug APK。

### 清理构建缓存

```bash
./gradlew clean
```

## 安装到设备

### 方法 1：USB 连接安装

1. 在 Android TV 上启用 **开发者选项** 和 **USB 调试**
2. 用 USB 线连接电视和电脑
3. 执行安装命令：

```bash
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

**参数说明**：
- `-r`：覆盖安装（如果已安装旧版本）
- 不加 `-r`：全新安装（已有版本会报错）

### 方法 2：Wi-Fi 无线安装（推荐）

#### 2.1 首次连接（需要 USB）

1. USB 连接设备
2. 执行：

```bash
adb tcpip 5555
```

3. 拔掉 USB 线
4. 获取电视 IP 地址（在电视的设置 → 网络中查看）
5. 连接设备：

```bash
adb connect 192.168.x.x:5555
```

将 `192.168.x.x` 替换为电视的实际 IP。

#### 2.2 安装 APK

```bash
adb -s 192.168.x.x:5555 install -r app/build/outputs/apk/debug/app-debug.apk
```

**示例**：

```bash
adb -s 192.168.2.116:5555 install -r app/build/outputs/apk/debug/app-debug.apk
```

#### 2.3 断开连接

```bash
adb disconnect 192.168.x.x:5555
```

### 方法 3：使用文件管理器安装

1. 将 APK 复制到 U 盘
2. U 盘插入 Android TV
3. 用电视的文件管理器打开 APK 安装

### 常见安装错误

#### 错误 1：签名冲突

```text
INSTALL_FAILED_UPDATE_INCOMPATIBLE
```

**解决方案**：先卸载旧版本

```bash
adb uninstall com.weekd.miracastreceiver
adb install app/build/outputs/apk/debug/app-debug.apk
```

#### 错误 2：未签名

```text
INSTALL_PARSE_FAILED_NO_CERTIFICATES
```

**原因**：安装了未签名的 Release APK

**解决方案**：使用 Debug APK 或配置签名

#### 错误 3：设备未连接

```text
error: no devices/emulators found
```

**解决方案**：检查 ADB 连接

```bash
adb devices
```

应该看到：

```
List of devices attached
192.168.2.116:5555    device
```

## ADB 日志调试

### 查看所有日志

```bash
adb logcat
```

**警告**：日志量巨大，不推荐直接查看。

### 过滤应用日志

只看本应用的日志：

```bash
adb logcat | grep "miracastreceiver"
```

### 过滤关键字

```bash
# 查看投屏相关日志
adb logcat | grep -i "DLNA\|AirPlay\|Miracast"

# 查看播放器日志
adb logcat | grep -i "PlayerActivity\|ExoPlayer"

# 查看错误日志
adb logcat | grep -E "E/|Error|Exception"
```

### 组合过滤（推荐）

```bash
adb logcat | grep -i "weekd\|miracastreceiver\|DLNA\|AirPlay\|PlayerActivity\|ExoPlayer"
```

### 清空日志缓冲

开始新的调试会话前，先清空旧日志：

```bash
adb logcat -c
```

### 保存日志到文件

```bash
adb logcat > logcat.txt
```

停止保存：按 `Ctrl+C`

### 实时查看并保存

```bash
adb logcat | tee logcat.txt
```

### 远程设备日志

指定设备 IP：

```bash
adb -s 192.168.2.116:5555 logcat | grep "miracastreceiver"
```

### 日志级别过滤

```bash
# 只看错误和警告
adb logcat *:E *:W

# 只看本应用的错误
adb logcat | grep -E "miracastreceiver.*E/"
```

## 获取报错信息

### 场景 1：投屏失败

**抓取日志**：

```bash
adb logcat -c
adb logcat | grep -i "SetAVTransportURI\|DLNA SetURI\|PlayerActivity\|Error\|Exception"
```

然后尝试投屏，复制出现的错误日志。

**关键信息**：
- `DLNA SetAVTransportURI: <URL>` - 投屏的媒体地址
- `PlayerActivity: Playing media: <URL>` - 播放器接收的地址
- `Error` / `Exception` - 错误堆栈

### 场景 2：播放器报错

**抓取日志**：

```bash
adb logcat | grep -i "ExoPlayerImplInternal\|PlaybackException\|PlayerActivity.*Error"
```

**典型错误示例**：

#### 错误 1：明文 HTTP 被拦截

```text
Cleartext HTTP traffic to xxx.xxx.xxx.xxx not permitted
```

**原因**：Android 9+ 默认禁止 HTTP
**已修复**：在 `AndroidManifest.xml` 添加 `android:usesCleartextTraffic="true"`

#### 错误 2：HLS 流无法播放

```text
UnrecognizedInputFormatException: None of the available extractors could read the stream
```

**原因**：缺少 HLS 支持库或未识别 HLS 格式
**已修复**：添加 `media3-exoplayer-hls` 并识别 `.m3u8`

#### 错误 3：视频格式不支持

```text
ClassNotFoundException: androidx.media3.exoplayer.hls.HlsMediaSource$Factory
```

**原因**：缺少 HLS 库
**已修复**：在 `build.gradle.kts` 添加依赖

### 场景 3：Miracast/WFD 无法发现

**抓取日志**：

```bash
adb logcat | grep -i "WifiDirect\|WFD\|Miracast\|P2P"
```

**检查项**：
- Wi-Fi Direct 是否启动
- WFD Info 是否设置成功
- 权限是否授予

**典型输出**：

```text
✅ WFD Info set successfully!
```

或：

```text
❌ SecurityException: setWFDInfo 需要系统权限
```

### 场景 4：应用崩溃

**抓取崩溃日志**：

```bash
adb logcat | grep -E "FATAL|AndroidRuntime"
```

或查看完整崩溃堆栈：

```bash
adb logcat *:E
```

### 场景 5：Emby 投屏 URI 为空

**抓取日志**：

```bash
adb logcat | grep -i "SetAVTransportURI\|Empty URI\|Request body"
```

**会看到**：

```text
SetAVTransportURI body (first 1000 chars): <完整 SOAP 请求>
Empty URI! Trying fallback parsing...
Request body: <完整请求体>
```

发送这些日志用于分析 Emby 的 SOAP 格式。

## 常见调试技巧

### 1. 查看应用进程

```bash
adb shell ps | grep miracastreceiver
```

### 2. 查看应用包信息

```bash
adb shell pm list packages | grep miracastreceiver
```

### 3. 强制停止应用

```bash
adb shell am force-stop com.weekd.miracastreceiver
```

### 4. 启动应用

```bash
adb shell am start -n com.weekd.miracastreceiver/.ui.MainActivity
```

### 5. 查看网络连接

```bash
adb shell netstat | grep 7000    # AirPlay 端口
adb shell netstat | grep 8080    # DLNA 端口
adb shell netstat | grep 7236    # WFD RTSP 端口
```

### 6. 查看设备信息

```bash
adb shell getprop | grep "ro.product"
```

### 7. 截图（调试界面问题）

```bash
adb shell screencap -p /sdcard/screenshot.png
adb pull /sdcard/screenshot.png
```

### 8. 录屏（调试播放问题）

```bash
adb shell screenrecord /sdcard/demo.mp4
# 操作完成后按 Ctrl+C 停止
adb pull /sdcard/demo.mp4
```

### 9. 查看 Wi-Fi Direct 状态

```bash
adb shell dumpsys wifip2p
```

### 10. 测试 RTSP 端口

从 PC 测试 WFD RTSP 端口是否开放：

```bash
telnet 192.168.2.116 7236
```

或：

```bash
nc -zv 192.168.2.116 7236
```

## Android Studio 调试

### 无线调试（Android Studio）

1. 打开 Android Studio
2. 点击 **Run** → **Edit Configurations**
3. 在 **Target** 选项中选择 **USB Device**
4. 通过 ADB 连接设备后，设备会自动出现在列表中

### 断点调试

1. 在代码中设置断点
2. 点击 **Debug** 按钮（绿色虫子图标）
3. 操作应用触发断点
4. 查看变量、堆栈等信息

### Logcat 窗口

Android Studio 底部有 **Logcat** 窗口，可以：
- 过滤应用包名
- 搜索关键字
- 设置日志级别
- 导出日志

## 故障排查流程

1. **复现问题** - 记录操作步骤
2. **清空日志** - `adb logcat -c`
3. **开始抓日志** - 使用上述过滤命令
4. **重现问题** - 按步骤操作
5. **停止抓日志** - `Ctrl+C`
6. **分析日志** - 查找 Error/Exception
7. **定位代码** - 根据堆栈信息找到出错位置
8. **修复问题**
9. **重新构建** - `./gradlew assembleDebug`
10. **重新安装** - `adb install -r ...`
11. **验证修复** - 重复测试
