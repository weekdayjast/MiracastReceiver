## 📝 技术决策记录

### 1. 选择 WebRTC 作为第一阶段协议
**原因**:
- 低延迟
- 跨平台支持好
- 有成熟的库和文档
- 可扩展性强

### 2. 使用 Kotlin 作为开发语言
**原因**:
- Android 官方推荐
- 代码简洁
- 协程支持好
- 更安全（空安全）

### 3. 使用 mDNS 进行服务发现
**原因**:
- 局域网自动发现
- 无需中心服务器
- Android 原生支持（NsdManager）
- 适合 P2P 场景

### 4. 最低 SDK 设置为 API 21
**原因**:
- 覆盖绝大多数 Android TV 设备
- 支持所需的 API（MediaCodec、NSD 等）
- Android 5.0+ 市场占有率高
